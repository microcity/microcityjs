# 实体编程说明

这是仿真实体的统一编程说明，所有仿真实体须按照此说明提供数据、接口

# 实体数据

每个仿真实体放置在单独的文件夹中，在此文件夹中存放自己的数据，有个三个默认文件（可以省略）按如下命名规则：
1. mesh.glb（或gltf） 实体三维模型数据
2. act.md 实体行为说明
3. act.js 实体行为脚本

**在act.md中须包含实体所有的状态集合、行为模式和指令集合及其含义**

## 实体接口

实体须提供如下接口以供调度器以及其他实体访问：
1. 获取当前实体三维模型的位置和旋转
2. 获取当前实体的状态
3. 下达指令，需要等待的函数调用（同步互动）须包装成消息（即提供返回值消息接口函数）
