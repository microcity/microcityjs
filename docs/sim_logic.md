# 仿真逻辑（业务模型与规则）
除了最底层的事件（Event）以外，离散事件仿真的单线程中存在两种高级对象，分为“智能体（Agent）”和“规划器（Planner）“两种。智能体可以感知环境并自主决策，规划器管理事件并可由事件触发、统筹规划全局并协调调度智能体，为防止阻塞，对象间需要等待的同步互动（函数调用）须包装成消息（即提供返回值消息接口函数）。

## 智能体结构
智能体具有实体占据空间，存在有限的状态和行为模式并可以接受有限的指令，有一定的自主决策能力，每个智能体须按照如下约定构建：
1. 每个智能体有实体三维模型数据（glb或者gltf文件）并提供接口获取当前的位置和旋转
2. 每个智能体定义有限的状态并提供接口获取当前的状态信息
3. 每个智能体有有限的行为模式和可接受指令并提供接口接受指令

## 规划器结构
规划器不占据空间，纯算法，管理事件并由事件触发（定时事件、智能体事件和其他调度器事件），规划全局，可以获取智能体状态并能向智能体发送指令。通常包含如下类型规划器：
1. 启动器（Initiator），在仿真运行开始设置场景、参数、智能体数量，并启动定时器等
2. 定时器（Timer），在仿真运行中通过定时事件循环自触发，例如重绘场景、到达事件等
3. 调度器（Scheduler），在特定事件发生时触发，例如分配任务、规划路径、解决智能体冲突或协作等
