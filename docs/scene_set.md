# 场景设置（案例模块与细节）
本仿真实例为一个自动化仓库的性能评估（空间利用率和出入库效率），包含两种静态实体：地面、货架；一种AGV智能体和六个规划器：启动器、帧采集器（定时器）、出/入库需求发生器（定时器）、任务分配器（调度器）、冲突解决器（定时器、调度器）。

## 地面
实体文件：/data/ground.gltf
空间信息：中心在原点在xz平面尺寸为10*10的矩形，划分为10*10个大小为1*1的网格。货架放置在地面上与网格四边对齐；AGV在地面上，移动时左右两边与网格两边对其。以网格xz为离散坐标，(5,-2),(5,-3),(5,-4)这三个位置为入库点，(5,2),(5,3)(5,4)这三个位置为出库点。

## 货架
实体文件：/data/shelf.glb
空间信息：底面大小为1*1，下部的空间可以容纳一个AGV，AGV将其轻微抬起就可以移动到别处。

## AGV智能体
实体文件：/data/agv.glb
有限状态：空闲、空载移动、满载移动、满载
可接收指令：移动到、装载、卸载、暂停移动、恢复移动，接收到的指令存储到列表中按次序执行，但是遇到”暂停移动“和”恢复移动“时须立刻执行

行为模式：
- 初始状态为空闲
- 当空闲的AGV接收到“移动到”指令时计算从当前点到终点的最短路径，然后按照1的速度运行，并将状态设置为“空载移动“，到达终点后状态为”空闲“
- 当空闲的AGV在货架下方并接收到指令”装载“时，略微抬起货架（货架绑定到AGV上，**用时1**），并将状态设置为”满载“
- 当满载的AGV接收到“移动到”指令时计算从当前点到终点的最短路径（避开有货架的地方），然后按照**速度1**运行，并将状态设置为“满载移动“，到达终点后状态为”满载“
- 当满载的AGV接收到”卸载“指令时，放下货架（货架与AGV解绑，**用时1**，如果在出库点则随后消失），并将状态设置为”空闲“
- AGV在移动的过程中需提供接口以便随时查询其前方待经过的数个网格位置，并可随时接收指令“暂停移动”和“恢复移动”并立刻执行

## 启动器
1. 加载地面，在地面上随机摆放货架（**占据40%网格**），随机摆放空闲AGV（**10个**）
2. 启动帧采集器、出/入库需求发生器、冲突解决器

## 帧采集器（定时器）
1. 采集所有AGV智能体的位置并发送信息给渲染端
2. 记录当前真实时间与仿真时间
3. 按照仿真速度（**5倍速**）以及真实时间间隔（与上一次做差）计算仿真时间间隔并预约下一次触发事件

## 出/入库需求发生器（定时器）
1. 产生一个出/入库需求（如果是入库需求将一个新货架随机放到入库点）
2. 触发任务分配器
3. 按照指数分布（**平均间隔1个时间单位**）计算时间间隔并预约下一次触发事件

## 任务分配器（调度器）
1. 读取需求
    - 如果是入库需求首先在仓库中找一个空位（尽量在现有货架旁边）
    - 如果是出库需求在仓库中找到一个货架（尽量找落单的或把头的）
2. 给任务最少并离装载货架处最近的AGV发送指令
    - 如果是入库需求则发送：移动到（入库位）、装载、移动到（预留空位）、卸载
    - 如果是出库需求则发送：移动到（出库货架）、装载、移动到（出库位）、卸载

## 冲突解决器（定时器、调度器）
1. 每隔1时间单位查询所有AGV未来**1个时间单位**待经过的网格位置
2. 标记所有有重叠的网格位置，只允许一个AGV移动，另外AGV发送“暂停移动“指令
3. 检查上一次已暂停移动的AGV前方网格位置是否被其他AGV重叠，如果没有向AGV发送“恢复移动”
